# qpdfImageEmbed

PDF utility that stamps images, QR codes, and text onto existing PDF pages using qpdf.

## Build

```bash
mkdir -p build && cd build && cmake .. && make
```

Dependencies (Debian/Ubuntu):
```bash
apt install libqpdf-dev libboost-program-options-dev libmagick++-dev libqrencode-dev cmake gcc
```

- C++20, single CMake project, single binary: `src/` → `qpdfImageEmbed`
- `install(TARGETS qpdfImageEmbed DESTINATION bin)` — `make install` to install

## Build quirks

- `config.h.in` is configured by CMake → generated into `build/src/config.h`
- `src/CMakeLists.txt` sets `MAGICKCORE_QUANTUM_DEPTH=16`, `MAGICKCORE_HDRI_ENABLE=0`, `POINTERHOLDER_TRANSITION=4` as compile definitions
- `.clang-format` present but no format script — run `clang-format -i src/*.cpp src/*.h` manually

## Architecture

- Entrypoint: `src/main.cpp` — parses CLI options, then processes the PDF
- `Options` → `readCLIOptions()` returns a `variant` map from `boost_program_options`
- `PDFProcessor` wraps `QPDF`, manages page geometry and XObject stream injection
- `ImageProvider` wraps both file-based images (Magick++) and programmatic QR codes (libqrencode)
- `ExtraText` parses `[x,y:size:style:]text` format strings
- `Rect` / `Point` — geometry helpers for page coordinates

## Tests (GTest via CTest)

Tests are off by default. Enable with `-DBUILD_TESTS=ON`:

```bash
cmake -S . -B build -DBUILD_TESTS=ON
cmake --build build
ctest --output-on-failure --test-dir build
```

- GTest v1.14 fetched via CMake FetchContent at configure time
- Five test executables under `tests/`: `test_point` (4), `test_rect` (7), `test_extraText` (14), `test_options` (10), `test_integration` (12) = **47 total**
- `test_extraText` and `test_integration` define `Logger logger;` locally — they do not link `main.cpp`
- `test_integration` links all project sources + qpdf/ImageMagick/boost/qrencode; has both class-level integration tests (ImageProvider, PDFProcessor) and CLI end-to-end tests (spawns the binary)
- `test_options` uses death tests for invalid CLI invocations; links boost_program_options
- Test assets live in `tests/data/`: `blank.pdf` (minimal valid PDF with /Resources), `test_image.ppm` (100×100 red image)

### Valgrind memcheck

Valgrind is configured via `CTestCustom.cmake.in` → generated `CTestCustom.cmake` in build dir. Run:

```bash
ctest -T memcheck --test-dir build
# or direct:
valgrind --leak-check=full --error-exitcode=1 build/tests/test_<name>
```

Note: `ctest -T memcheck` requires `DartConfiguration.tcl` (generated by `include(CTest)`). If it fails, run valgrind directly as shown above.
